# Vision del producto

## Proposito
Construir una webapp simple y rapida para gestionar clases de gimnasio por reservas semanales, reemplazando la gestion manual.

## Problema actual
- Inscripciones y cambios se hacen a mano.
- Dificultad para controlar aforos y lista de espera.
- Falta de visibilidad del bono y asistencias para el alumno.
- El administrador pierde tiempo en tareas repetitivas.

## Objetivos de negocio
- Mejorar la ocupacion de clases con reservas claras.
- Reducir el tiempo de gestion del administrador.
- Mejorar la experiencia del alumno con informacion y autonomia.

## Principios de producto
- Mobile first para alumnos y monitores.
- Flujos cortos: reservar, cambiar, cancelar.
- Datos visibles y confiables (bono, sesiones restantes, historial).
- Transparencia en penalizaciones por cancelacion tardia.

## Usuarios objetivo
- Alumnos con bonos mensuales y necesidad de autogestion.
- Monitores que gestionan sesiones y ejercicios.
- Administrador que opera el sistema y verifica asistencia.

## Exito esperado (metricas sugeridas)
- 90% de reservas hechas sin intervencion del admin.
- Disminucion del tiempo de gestion semanal.
- Menos errores en aforo y asistencia.

## Restricciones
- Sin pagos integrados.
- Horarios mayormente fijos, con cambios puntuales.
- Uso de tecnologias web basicas por simplicidad.
